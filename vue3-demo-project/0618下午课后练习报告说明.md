## 4. 怎么看练习 2 的结果：Vite 多代理规则

练习 2 要验证的是：`vite.config.js` 中可以配置多条代理规则，并且不同前缀会转发到不同服务器。

页面操作：

1. 打开浏览器页面。
2. 按 `F12` 打开开发者工具。
3. 切换到 `Network` 面板。
4. 点击页面中的 `测试 /api-user 代理` 按钮。
5. 再点击页面中的 `测试 /api-skill 代理` 按钮。

预期结果：

1. `Network` 面板中能看到 `/api-user/profile` 请求。
2. `Network` 面板中能看到 `/api-skill/list` 请求。
3. 因为课件要求中使用的是虚拟服务器 `http://user.server.com` 和 `http://skill.server.com`，所以请求失败是正常的。
4. 本练习重点不是让虚拟服务器返回真实数据，而是证明 Vite 代理规则已经配置并能匹配对应前缀。

对应代码位置：

```text
vite.config.js
```

关键代码：

```js
'/api-user': {
  target: 'http://user.server.com',
  changeOrigin: true,
  rewrite: (path) => path.replace(/^\/api-user/, ''),
},

'/api-skill': {
  target: 'http://skill.server.com',
  changeOrigin: true,
  rewrite: (path) => path.replace(/^\/api-skill/, ''),
},
```

路径重写含义：

```text
/api-user/profile  →  http://user.server.com/profile
/api-skill/list    →  http://skill.server.com/list
```
